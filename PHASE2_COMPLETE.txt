ENVIRONMENT CONFIGURATION
================================================================================

Required in .env:

  NEXT_PUBLIC_SENTRY_DSN              # Client-side DSN
  SENTRY_ENVIRONMENT                  # development/staging/production
  NEXT_PUBLIC_SENTRY_ENVIRONMENT      # Client environment
  SENTRY_TRACES_SAMPLE_RATE          # 0.0-1.0 (1.0 for dev, 0.1 for prod)
  NEXT_PUBLIC_SENTRY_TRACES_SAMPLE_RATE # Client sample rate

Optional:
  SENTRY_RELEASE                      # Version/commit hash
  NEXT_PUBLIC_SENTRY_RELEASE          # Client version

Without DSN: Sentry gracefully disables itself (no errors)

   NEXT_PUBLIC_SENTRY_DSN="https://..."
   SENTRY_ENVIRONMENT="development"

================================================================================
KNOWN LIMITATIONS & CONSIDERATIONS
================================================================================

1. Performance
   - Production uses 10% sample rate to reduce costs
   - Some errors may not be captured in production

2. Network
   - Requires internet connectivity to send to Sentry
   - Events queued locally if offline (temporary)

3. Storage
   - Session replay disabled by default (privacy)
   - Can be enabled for production if desired

4. Local Development
   - No events sent if DSN is empty (by design)
   - Disables Sentry gracefully

5. Edge Runtime
   - Limited Sentry support (core features only)
   - Primarily supports Node.js runtime
